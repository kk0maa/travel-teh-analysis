## Бизнес-контекст

**Профиль компании:** Тревел-тех платформа, агрегатор онлайн-бронирования билетов и отелей.  
**Целевая аудитория:** Широкая, сегментирована на молодежь и взрослую аудиторию.  
**Программа лояльности:** Пользователи делятся на обычных и VIP с приоритетной поддержкой и кэшбеком.

**Стратегические цели бизнеса:**
- Увеличить GMV на 40% за счет мобильного приложения
- Снизить долю отмен бронирований на 5%

## Система продуктовых метрик

### North Star Metric (NSM)
**GMV (Gross Merchandise Value)** — общий объем оплаченных бронирований.

### Ключевые метрики продукта

| Метрика | Описание | Назначение |
|---------|----------|------------|
| **Conversion Rate (CR)** | % перехода от поиска к успешной оплате | Оценка воронки продаж |
| **Доля VIP** | % пользователей с премиум-статусом | Стабильная база выручки |
| **ARPU** | Средний доход с доп. услуг (страховки, багаж) | Маржинальность чека |
| **CAC** | Стоимость привлечения пользователя | Эффективность маркетинга |
| **Cancellation Rate** | Доля отмененных бронирований | Операционные издержки |

## Описание данных

### Признаковое пространство

| Переменная | Тип | Описание |
|------------|-----|----------|
| `action_type` | category | Этап воронки (entry, good, bad) |
| `action_dttm` | datetime | Дата совершения действия |
| `revenue_v1` | float | Оборот от бронирований |
| `revenue_v2` | float | Доход с дополнительных услуг |
| `cost_v1` | float | Издержки на привлечение |
| `cost_v2` | float | Убытки от отмен |
| `x5` | binary | Платформа (1 — приложение, 0 — сайт) |
| `x4` | category | Направление (S, W, E, N) |
| `x1` | binary | Статус пользователя (1 — VIP, 0 — обычный) |
| `x3` | int | Возраст клиента |

## Методология исследования

### Проверенные гипотезы

1. **Гипотеза по CR:** Конверсия через приложение выше, чем через сайт, за счет привязки карт и покупки в 1 клик.
   - **Зачем бизнесу:** Перераспределить бюджеты на рекламу мобильного приложения.

2. **Гипотеза по NSM (сезонность):** Направления "Юг" и "Запад" генерируют основной поток бронирований весной и летом.
   - **Зачем бизнесу:** Сфокусировать закупки отелей/авиалиний для получения скидок за объем.

3. **Гипотеза по ARPU:** Участники лояльности (VIP) покупают больше доп. услуг, чем обычные клиенты.
   - **Зачем бизнесу:** Обосновать затраты на содержание программы лояльности.

4. **Гипотеза по отменам:** Молодая аудитория (до 25 лет) чаще отменяет бронирования из-за спонтанного планирования.
   - **Зачем бизнесу:** Ввести тарифы "без отмены" или комиссии для этого сегмента.

5. **Гипотеза по CAC:** Стоимость привлечения на непопулярных направлениях выше, чем на классических.
   - **Зачем бизнесу:** Оптимизировать ставки в рекламе.

## Результаты анализа

### Главные тенденции

#### Динамика оборота по направлениям
- **Север:** Стабильная динамика (11,000 – 52,000 рублей)
- **Восток и Запад:** Рост оборота в ноябре на **110%** и **130%** соответственно

#### Конверсия по платформам
| Платформа | Conversion Rate |
|-----------|-----------------|
| Мобильное приложение | **67%** |
| Веб-сайт | 62% |

**Вывод:** Приложение демонстрирует конверсию на **5 процентных пунктов выше**.

#### Доля отмен по возрастным группам
- **Приложение:** Самая высокая доля отмен у молодежи **15–20 лет**
- **Сайт:** Самая высокая доля отмен у пожилых пользователей **60+ лет**

#### Динамика ARPU
- Средний чек доп. услуг в 2026 году снизился:
  - **Стандарт:** на 5% относительно 2025 года
  - **VIP:** на 2% относительно 2025 года

#### Cancellation Rate
- Доля отмен в ноябре 2026: **11.9%** (на **4.6% меньше** относительно марта 2025)

## Бизнес-рекомендации

### 1. Развитие мобильного приложения
**Действие:** Перенаправить рекламный бюджет на кампании по установке мобильного приложения.  
**Обоснование:** Конверсия приложения на 5 п.п. выше сайта (67% vs 62%).

### 2. Смена приоритетов контрактования
**Действие:** Сместить фокус закупок с Юга на **Восток и Запад**.  
**Обоснование:** Рост оборота на этих направлениях составил 110–130% в ноябре.

### 3. Реструктуризация VIP-программы
**Действие:** Разработать новые премиальные услуги для VIP-пакета.  
**Обоснование:** ARPU VIP-клиентов на 1.8% ниже обычных — требуется пересмотр ценностного предложения.

### 4. Управление отменами
**Действие:** Бороться с отменами продуктовые методами для всей базы пользователей.  
**Обоснование:** Гипотеза о молодежных отменах не подтвердилась — проблема системная.

### 5. Сглаживание сезонности
**Действие:** Внедрить акции «раннего бронирования» осенью.  
**Обоснование:** Снижение зависимости от сезонных пиков.

### 6. Оптимизация CAC
**Действие:** Снижать стоимость клика для направления Север, распределять бюджет на Восток и Запад.  
**Обоснование:** CAC равномерен по всем направлениям (~20), но ROI разный.
